# Prise en charge du protocole DocumentDB pour MongoDB

Les bases de données Microsoft Azure DocumentDB peuvent désormais être utilisées comme magasins de données pour les applications écrites pour MongoDB. À l’aide de pilotes existants pour MongoDB et de cette fonctionnalité en version préliminaire, les applications écrites pour MongoDB peuvent communiquer facilement et en toute transparence avec DocumentDB. Les développeurs peuvent utiliser leurs compétences et outils actuels relatifs à MongoDB pour créer des applications qui tirent parti des bases de données NoSQL complètement gérées et évolutives de DocumentDB.

Nous proposons actuellement à nos clients de découvrir cette nouvelle fonctionnalité en version préliminaire. Si vous souhaitez également la découvrir, suivez les instructions ci-dessous pour demander l’accès au programme d’évaluation. Nous vous contacterons dès que vous serez sélectionné.

## Composants requis

Avant de demander l’accès au programme d’évaluation, vous devez disposer des éléments suivants.

- **Un abonnement Azure**. Voir [Créez votre compte Azure gratuit](https://azure.microsoft.com/free/).

## Demander l’accès à la fonctionnalité en version préliminaire de prise en charge du protocole de DocumentDB pour MongoDB  

Pour demander l’accès à la version préliminaire, procédez comme suit.

1. Connectez-vous au [portail Azure](https://portal.azure.com).
2. Cliquez sur **Nouveau**, puis sur **Données et stockage**, **Afficher tout**, **Plus** et enfin sur **DocumentDB - Prise en charge de protocole pour MongoDB**.

	![Capture d’écran des panneaux Marketplace et Données et stockage, mise en surbrillance de DocumentDB - Prise en charge de protocole pour MongoDB, Base de données Mongo](./media/documentdb-protocol-mongodb/marketplacegallery1.png)

3. Vous pouvez également rechercher « Protocole DocumentDB » dans la catégorie **Données et stockage**.

	![Capture d’écran des panneaux Marketplace et recherche des données et du stockage, mise en surbrillance de DocumentDB - Prise en charge de protocole pour MongoDB, Base de données Mongo](./media/documentdb-protocol-mongodb/marketplacegallery2.png)

4. Cliquez sur **Créer** pour lancer le processus d’inscription à la version préliminaire.

	![Le panneau DocumentDB - Prise en charge de protocole pour MongoDB dans le portail Azure](./media/documentdb-protocol-mongodb/marketplacegallery3.png)

5. Dans le panneau **Nouveau compte DocumentDB**, cliquez sur **S’inscrire à la version préliminaire**. Lisez les informations, puis cliquez sur **OK**. Vous recevez un courrier électronique une fois votre abonnement activé pour la version d'évaluation publique.

	![Le panneau S’inscrire à la version préliminaire pour DocumentDB - Prise en charge de protocole pour MongoDB dans le portail Azure](./media/documentdb-protocol-mongodb/registerforpreview.png)


## Étapes suivantes
- Une fois que vous avez été sélectionné pour la version préliminaire, vous recevrez un message de bienvenue contenant les instructions nécessaires à la création de votre compte et à la prise en main de celui-ci.
