Pour le développement local dans Azure Functions version2.x, le package est utilisé pour [inscrire l’extension de liaison](../articles/azure-functions/functions-triggers-bindings.md#local-development-azure-functions-core-tools). Pour le développement d’une bibliothèque de classes C# dans Functions 1.x et 2.x, la prise en charge des liaisons peut être ajoutée par [l’installation du package](../articles/azure-functions/functions-triggers-bindings.md#local-c-development-using-visual-studio-or-vs-code).

Notez que le code source suit les numéros de version du SDK WebJobs. Le SDK WebJobs version 2.x correspond à Azure Functions 1.x. Le code de Functions 1.x se trouve donc dans la branche v2.x du référentiel. Les branches Master et Dev contiennent du code pour WebJobs 3.x, qui correspond à Functions 2.x.
