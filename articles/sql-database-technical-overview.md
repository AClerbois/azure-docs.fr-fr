# Vue d’ensemble des bases de données SQL

Une base de données SQL est un service de base de données relationnelle dans le cloud, qui repose sur le moteur Microsoft SQL Server et propose des fonctionnalités stratégiques intégrées. Une base de données SQL offre des fonctions prévisibles de gestion des performances, de l’évolutivité, de la continuité d’activité et de la protection des données, ainsi qu’une administration proche de zéro aux développeurs du cloud et aux architectes de solutions. Mieux encore, la base de données SQL prend en charge les outils, bibliothèques et API SQL Server existants, ce qui facilite votre transition vers le cloud. Votre base de données SQL peut être [opérationnelle en quelques secondes](sql-database-get-started.md), sans que vous soyez contraint de gérer une infrastructure ou des machines virtuelles. Les fonctionnalités de programmation de type administrateur de bases de données permettent des opérations de développement efficaces, alors que la fonction d’autogestion assure une maintenance proche de zéro pour les bases de données. Ainsi, vous pouvez vous concentrer sur le développement et la gestion de votre application.

La base de données SQL vous permet d’assurer la [montée en puissance](sql-database-service-tiers.md) ou la [montée en charge](sql-database-elastic-scale-get-started.md) de l’infrastructure, de manière à inclure des milliers de bases de données, tout en bénéficiant de performances prévisibles pouvant être augmentées ou réduites en fonction des besoins. Grâce à un [contrat SLA](http://azure.microsoft.com/support/legal/sla/) proposant une disponibilité de 99,99 %, leader sur le marché, vos applications bénéficient d’une probabilité de coupure de service proche de zéro. La fonction de protection des données intégrée, qui repose sur l’audit, la restauration et la géo-réplication, offre une préservation continue de vos données. La base de données SQL vous permet de restaurer vos données à n’importe quel moment, à partir de n’importe quelle transaction, sur une période de 35 jours maximum. Vous pouvez également répliquer vos données dans la région Microsoft Azure de votre choix et implémenter aisément une stratégie de récupération d’urgence géographique, conçue pour répondre aux besoins de votre entreprise. La [dernière version de la base de données SQL](sql-database-preview-whats-new.md), la version 12, offre une compatibilité presque totale avec le moteur Microsoft SQL Server.

Il existe différents [niveaux de service](sql-database-service-tiers.md) pour la base de données SQL : le niveau de base, le niveau Standard et le niveau Premium. Ces niveaux prennent en charge des charges de travail de base de données légères ou lourdes, selon le cas. Vous pouvez ainsi passer d’un niveau à l’autre ou les associer les uns aux autres, afin de bénéficier d’une conception d’applications innovante. Grâce à la puissance et à la portée de Microsoft Azure, vous pouvez mélanger les différents services Microsoft Azure avec la base de données SQL, afin de pouvoir répondre aux besoins professionnels uniques de vos nouvelles applications, réduire les coûts, optimiser l’efficacité et mettre en lumière de nouvelles opportunités professionnelles.

Obtenez une version d’[essai gratuit](http://azure.microsoft.com/pricing/free-trial/) pour prendre en main cette nouvelle technologie, et découvrez [comment créer votre première base de données SQL](sql-database-get-started.md)!
