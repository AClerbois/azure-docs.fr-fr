# <a name="azure-vmware-solution-upgrade-concepts"></a>Concepts des mises à niveau d’Azure VMware Solution

L’un des principaux avantages des clouds privés d’Azure VMware Solution réside dans le fait que la plateforme est tenue à jour pour vous. La maintenance de la plateforme comprend les mises à niveau automatisées vers une offre groupée de logiciels validée VMware. Les mises à niveau sont effectuées régulièrement, vous garantissant ainsi de toujours disposer des dernières versions validées des logiciels.

## <a name="azure-vmware-solution-private-cloud-software-upgrades"></a>Mises à jour logicielles du cloud Azure VMware Solution

La plateforme de cloud privé Azure VMware Solution comprend des versions spécifiques des logiciels VMware vSphere, ESXi, vSAN et NSX-T. L’offre groupée de logiciels de cloud privé est validée pour une utilisation dans de nouvelles installations de cloud privé, et pour les mises à niveau de clouds privés existants.

Le processus de mise à niveau de la gestion du cycle de vie ne nécessite aucun temps d’arrêt de vos clouds privés. Le processus de mise à niveau garantit que vous utilisez automatiquement la dernière version en date des logiciels de cloud privé Azure VMware Solution validés. Les mises à niveau sont appliquées à un rythme régulier, afin que les versions des clouds privés ne soient jamais plus anciennes que d’une version par rapport à la toute dernière version de l’offre groupée de logiciels validés. Vous serez averti des mises à niveau planifiées sur votre cloud privé. Vous pouvez différer la mise à niveau si la version de votre cloud privé est antérieure d’une version par rapport à la version la plus récente.

Les correctifs et mises à jour critiques sont appliqués lorsqu’ils sont validés. Vous serez averti à l’avance des mises à niveau critiques obligatoires. Cette stratégie garantit que votre cloud privé bénéficie de l’application immédiate des correctifs et mises à jour critiques.

Les versions logicielles de VMware sont fournies dans [l’article sur les concepts des clusters et des clouds privés](concepts-private-clouds-clusters.md) et dans les [Questions fréquentes (FAQ)](faq.md).
