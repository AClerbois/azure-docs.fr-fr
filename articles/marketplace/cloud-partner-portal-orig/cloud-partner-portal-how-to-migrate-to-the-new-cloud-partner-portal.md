# <a name="how-to-migrate-to-the-new-cloud-partner-portal"></a>Guide pratique pour migrer vers le nouveau portail Cloud Partner

La migration vers le portail Cloud Partner est une étape importante. En réponse à vos commentaires sur le processus de publication, nous avons investi dans le portail. Ces investissements visaient à en faciliter l’utilisation et à améliorer l’expérience de publication. 

-   Support pour les connexions d’ID de société
-   Validation de champ en temps réel pour détecter les erreurs
-   Workflow de publication simplifié

Nous avons facilité la migration de vos offres. Par conséquent, si vous êtes prêt pour une meilleure expérience de publication, passez en revue les étapes de migration suivantes.

Pour réussir votre migration, passez en revue les étapes de migration suivantes.



## <a name="decide-youre-ready-to-migrate"></a>Décider si vous êtes prêt pour la migration

Avant de procéder à la migration de vos offres vers le portail Cloud Partner, assurez-vous que vous comprenez parfaitement ce qui va se produire.

**Suis-je en train de migrer toutes mes offres ?**

Nous migrons actuellement toutes les offres de machine virtuelle. Toutes vos offres de machine virtuelle (brouillon, version intermédiaire ou répertoriée) seront migrées.

Tous les autres types d’offres (modèles de solution, versions d’évaluation,etc.) continueront à fonctionner sur l’ancien portail pour le moment.

**Mon offre en ligne continuera-t-elle de fonctionner ?**

Oui, cette migration n’affectera pas vos offres en ligne. Nous avons également prérempli les informations de votre compte d’éditeur pour vous permettre de mettre immédiatement à jour votre offre sur le nouveau portail une fois la migration terminée. Les offres intermédiaires deviendront des brouillons et vous devrez les publier sur le nouveau portail.

**Lors de la migration, puis-je changer la tarification de mes offres ?**

Non, pour l’ensemble de vos offres en ligne, la tarification doit rester identique, car nous ne prenons pas en charge les modifications de prix dans cette migration.

**Que se passe-t-il si mon offre se trouve à l’état intermédiaire/publication ?**

Votre migration échouera si une de vos offres est actuellement publiée ou en version intermédiaire sur l’ancien portail. Lorsque vous choisissez l’heure de la migration, il est important de vérifier que vos offres se trouvent toutes dans l’état publié/intermédiaire.

**Comment le nouvel accès en fonction du rôle fonctionne-t-il ?**

Dans le nouveau portail, votre administrateur devient le propriétaire du profil de publication et tous les coadministrateurs deviennent des collaborateurs. Si vous souhaitez modifier quoi que ce soit dans votre profil de publication, vous devez utiliser le nouveau portail.

Pour gérer vos utilisateurs, si vous souhaitez ajouter une personne qui travaillera sur une offre sur l’ancien portail, ajoutez-la toujours comme un coadministrateur sur l’ancien portail. Si vous souhaitez ajouter une personne qui travaillera sur une offre sur le nouveau portail, vous l’ajouterez en tant que collaborateur sur le nouveau portail.

**Ma migration est-elle finale ?**

Lorsque nous effectuons la migration de vos offres, toutes les actions effectuées sur l’ancien portail sont en lecture seule et bloquées sur l’ancien portail. Vous pouvez continuer à modifier et à développer toutes vos offres migrées là où vous les aviez laissées, mais uniquement dans le nouveau portail.

## <a name="submit-a-request-to-migrate"></a>Envoyer une demande de migration

Une fois que vous avez déterminé que vous êtes prêt à planifier votre migration, soumettez votre migration sur la page Learn du portail Cloud Partner avec votre nom et votre ID d’éditeur.

![Diagramme du processus de migration](./media/cloud-partner-portal-how-to-migrate-to-the-new-cloud-partner-portal/migrate.png)

## <a name="confirm-a-date-to-migrate"></a>Confirmer la date de la migration

Nous effectuons la migration des offres toutes les semaines le mardi de 10h à 13h PST, et demandons au moins 24 heures à l’avance au propriétaire du compte de publication de confirmer l’heure de la migration. Si vous devez effectuer la migration à une autre heure afin que l’opération n’interfère pas avec votre workflow, nous rechercherons ensemble le moment le plus propice.

Vous recevrez une notification par e-mail qui vous informera que la migration est terminée et pourrez alors continuer à travailler sur le nouveau portail.

## <a name="settle-in-and-enjoy"></a>Vous êtes prêt ? C’est parti !

Après la migration, vous devriez être opérationnel sur le nouveau portail. Désormais, vos offres apparaîtront en lecture seule sur l’ancien portail et toutes les futures créations et modifications devraient provenir du nouveau portail !

**Un problème est survenu pendant ma migration, que dois-je faire ?**
 
Si votre migration échoue, nous annulerons les modifications apportées et vous proposerons une autre heure de migration. N’hésitez pas à nous contacter via l’outil de commentaires si vous avez des questions, une fois votre migration terminée.
