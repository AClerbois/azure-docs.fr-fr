# <a name="glossary"></a>Glossaire

## <a name="a"></a>A

### <a name="acoustic-model"></a>Modèle acoustique

Le modèle acoustique est un classifieur qui étiquette de courts fragments audio en un certain nombre de phonèmes, ou unités sonores, dans une langue donnée. Par exemple, le mot « vocal » comprend cinq phonèmes : « v », « o », « k », « a » et « l ». Ces classifications sont effectuées une centaine de fois par seconde

## <a name="b"></a>b

## <a name="c"></a>C

### <a name="conversational-model"></a>Modèle de conversation

Un modèle approprié pour la reconnaissance vocale de paroles prononcées dans un style oral. Le modèle acoustique de conversation Microsoft est adapté pour les discours généralement destinés à une autre personne.

## <a name="d"></a>D

### <a name="deployment"></a>Déploiement

Le processus via lequel le modèle personnalisé adapté devient un service et expose un URI

### <a name="inverse-text-normalization"></a>Normalisation du texte inversée

Le processus de conversion de texte « brut » non formaté en texte formaté (avec majuscules et ponctuation, par exemple), est appelé normalisation du texte inversée.

## <a name="k"></a>K

## <a name="l"></a>L

### <a name="language-model"></a>Modèle de langage

Le modèle de langage est une distribution de probabilités sur des séquences de mots. Le modèle de langage aide le système à opérer un choix parmi des séquences de mots homophones, en se basant sur la probabilité des séquences de mots elles-mêmes

## <a name="m"></a>M

## <a name="n"></a>N

### <a name="normalization"></a>Normalisation

Normalisation (texte) : transformation du texte résultant (c'est-à-dire transcription) dans un format standard et non ambigu pouvant être lu par le système.

## <a name="s"></a>S

### <a name="search-and-dictate-model"></a>Modèle Recherche & Dictée

Un modèle acoustique approprié pour le traitement des commandes. Le modèle acoustique Recherche & Dictée de Microsoft est approprié pour les discours destinés à une application ou un appareil, comme pour des commandes.

### <a name="subscription-key"></a>Clé d’abonnement

La clé d’abonnement est une chaîne que vous devez spécifier comme paramètre de chaîne de requête afin d’appeler un modèle Custom Speech Service. Une clé d’abonnement est obtenue à partir du [Portail Azure](https://portal.azure.com/#create/Microsoft.CognitiveServices/apitype/CustomSpeech). Une fois que vous l’avez, vous pouvez la retrouver dans « Mes abonnements » dans le portail Custom Speech Service.

## <a name="t"></a>T

### <a name="transcription"></a>Transcription

Transcription : l’élément de texte qui résulte du traitement d’une partie du fichier audio .wav
