# <a name="how-to-migrate-to-the-new-cloud-partner-portal"></a>Guide pratique pour migrer vers le nouveau portail Cloud Partner

La migration vers le portail Cloud Partner est un grand pas en avant. Nous avons tenu compte de vos commentaires concernant le processus de publication et avons mis tous nos efforts pour améliorer la convivialité et l’expérience de publication dans son ensemble. 
*    Support pour les connexions d’ID de société
*    Validation de champ en temps réel pour détecter les erreurs
*    Workflow de publication simplifié

Nous avons tout mis en œuvre pour que la migration de votre offre soit très facile. Par conséquent, si vous souhaitez profiter d’une meilleure expérience de publication, voici ce que vous devez faire.

## <a name="what-do-you-need-to-do"></a>Que devez-vous faire ?
Réussissez votre migration en procédant comme suit :

**ÉTAPES DE LA MIGRATION**
|          |          |
|----------|----------|
|1.    Vérifier si vous êtes prêt | Avant de migrer vos offres vers le portail Cloud Partner, assurez-vous que vous êtes parfaitement conscient de ce qui va se produire.<br/><br/>**Suis-je en train de migrer toutes mes offres ?**<br/>Nous migrons actuellement toutes les offres de machine virtuelle. Cela signifie que toutes vos offres de machine virtuelle (brouillon, version intermédiaire ou répertoriée) seront migrées. <br/>Pour tout autre type d’offre (modèles de solution, versions d’évaluation,etc.), elles continueront à fonctionner sur l’ancien portail pour le moment.<br/><br/>**Mon offre en ligne continuera-t-elle de fonctionner ?**<br/>Oui, cette migration n’affectera pas vos offres en ligne. Nous avons également prérempli les informations de votre compte d’éditeur pour vous permettre de mettre immédiatement à jour votre offre sur le nouveau portail une fois que vous avez terminé la migration. Les offres intermédiaires deviendront des brouillons et vous devrez les publier sur le nouveau portail.<br/><br/>**Lors de la migration, puis-je changer la tarification de mes offres ?**<br/>Non, pour l’ensemble de vos offres en ligne, la tarification doit rester exactement identique car nous ne prenons pas en charge les modifications de prix dans cette migration.<br/><br/>**Que se passe-t-il si mon offre se trouve à l’état intermédiaire/publication ?**<br/>Votre migration échouera si une de vos offres est actuellement publiée ou en version intermédiaire sur l’ancien portail. Lorsque vous choisissez l’heure de la migration, il est important de vérifier que vos offres se trouvent toutes à l’état publié/intermédiaire.<br/><br/>**Comment le nouvel accès en fonction du rôle fonctionne-t-il ?**<br/>Dans le nouveau portail, votre administrateur devient le propriétaire du profil de publication et tous les coadministrateurs deviennent des collaborateurs. Si vous souhaitez modifier quoi que ce soit dans votre profil de publication, vous devez utiliser le nouveau portail. <br/>Pour gérer vos utilisateurs, si vous souhaitez ajouter une personne qui travaillera sur une offre sur l’ancien portail, ajoutez-la toujours comme un coadministrateur sur l’ancien portail. Si vous souhaitez ajouter une personne qui travaillera sur une offre sur le nouveau portail, vous l’ajouterez en tant que collaborateur sur le nouveau portail.<br/><br/>**Ma migration est-elle finale ?**<br/>Lorsque nous migrons vos offres, toutes vos actions effectuées sur l’ancien portail sont en lecture seule et bloquées via l’ancien portail. Vous pouvez continuer à modifier et à développer toutes vos offres migrées là où vous les aviez laissées, mais uniquement dans le nouveau portail.|
|2.    Soumettre une migration | Une fois que vous avez déterminé que vous êtes prêt à planifier votre migration, soumettez votre migration sur la page Learn du portail Cloud Partner en utilisant votre nom et votre ID d’éditeur.![Diagramme de flux de publication](./media/cloud-partner-portal-how-to-migrate-to-the-new-cloud-partner-portal/migrate.png)|
|3.    Confirmer une date | Nous migrons les offres chaque semaine le mardi de 10h à 13h PST, et demandons au moins 24 heures à l’avance au propriétaire du compte de publication de confirmer l’heure de la migration. Si vous devez effectuer la migration à une autre heure afin que l’opération n’interfère pas avec votre workflow, nous rechercherons ensemble le moment le plus propice.<br/>Vous recevrez une notification par e-mail qui vous informera que la migration est terminée et pourrez alors continuer à travailler sur le nouveau portail.|
|4.    Vous êtes prêt ? C’est parti ! | Après la migration, vous devriez être opérationnel sur le nouveau portail. Désormais, vos offres apparaîtront en lecture seule sur l’ancien portail et toutes les futures créations et modifications devraient provenir du nouveau portail ! <br/><br/>**Un problème est survenu pendant ma migration, que dois-je faire ?** Si votre migration échoue, nous annulerons les modifications apportées et vous proposerons une autre heure de migration. N’hésitez pas à nous contacter via l’outil de commentaires si vous avez des questions, une fois votre migration terminée.|
