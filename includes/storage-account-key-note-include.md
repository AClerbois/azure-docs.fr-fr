> [!IMPORTANT]
> Votre clé de compte de stockage est similaire au mot de passe racine pour votre compte de stockage. Veillez toujours à protéger votre clé de compte. Évitez de la communiquer à d’autres utilisateurs, de la coder en dur ou de l’enregistrer en texte brut dans un emplacement accessible à d’autres personnes. Régénérez votre clé de compte à l’aide du Portail Azure si vous pensez que sa confidentialité est compromise.
> 
> Lorsque cela est possible, Microsoft recommande d’utiliser l’authentification Azure Active Directory (Azure AD) pour vos applications de stockage d’objets blob et de file d’attente (préversion) pour une sécurité renforcée. Pour plus d’informations, consultez [Authentifier l’accès aux objets blob et files d’attente Azure à l’aide d’Azure Active Directory (préversion)](https://docs.microsoft.com/azure/storage/common/storage-auth-aad).
