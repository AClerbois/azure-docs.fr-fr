Cette rubrique inclut des exemples utilisant des applets de commande Azure PowerShell. Azure PowerShell est actuellement disponible en deux versions : 1.0 et 0.9.8. Si vous disposez de scripts et que vous ne souhaitez pas les modifier maintenant, vous pouvez continuer à utiliser la version 0.9.8. Lorsque vous utilisez la version 1.0, vous devez soigneusement tester vos scripts dans des environnements de pré-production avant de les utiliser en production pour éviter un impact inattendu.

Dans la plupart des cas, la seule différence entre les deux versions est que le nom des applets de commande de la version 1.0 suit le modèle {verbe}-AzureRm{nom}, tandis que dans la version 0.9.8, le nom n’inclut pas **Rm** (par exemple, New-AzureRmResourceGroup au lieu de New-AzureResourceGroup). Lorsque la différence entre les versions est plus importante, cette rubrique présente des exemples pour les deux versions.

Lorsque vous utilisez Azure PowerShell 0.9.8, vous devez d’abord activer le mode Resource Manager en exécutant la commande **Switch-AzureMode AzureResourceManager**. Cette commande n’est pas nécessaire dans la version 1.0.

Pour plus d’informations sur la version 1.0, notamment sur la procédure d’installation et de désinstallation de la version, consultez [Version préliminaire d’Azure PowerShell 1.0](https://azure.microsoft.com/blog/azps-1-0-pre/). Pour plus d’informations sur les modifications importantes apportées aux commandes Resource Manager, consultez [Modifications apportées aux applets de commande PowerShell Azure Resource Manager](../articles/powershell-preview-resource-manager-changes.md).
