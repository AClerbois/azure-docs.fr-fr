> [!TIP]
> Lorsque vous migrez des bases de données vers Azure à l’aide d’Azure Database Migration Service, vous pouvez opter pour une migration *hors connexion* ou *en ligne*. Dans le cas d’une migration hors connexion, le temps d’arrêt de l’application commence en même temps que le lancement de la migration. Dans le cas d’une migration en ligne, le temps d’arrêt est limité à la durée du basculement à la fin de la migration. Nous vous suggérons de tester une migration hors connexion pour déterminer si le temps d’arrêt est acceptable ; dans le cas contraire, privilégiez une migration en ligne.
