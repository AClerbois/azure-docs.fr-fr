# <a name="mainframe-workloads-supported-on-azure"></a>Les charges de travail mainframe pris en charge sur Azure

Les solutions répertoriées ici peuvent vous aider à migrer votre charge de travail mainframe vers Microsoft Azure. Certaines charges de travail peuvent être migrées assez facilement. Autres charges de travail qui dépendent de logiciels du système hérité peuvent être réhébergés. 

Pour plus d’aide avec l’émulation de grands systèmes et services, reportez-vous à la [centre de Migration de macroordinateurs Azure](https://azure.microsoft.com/migration/mainframe/).

## <a name="migrate-mainframe-closer-to-azure"></a>Migrer le plus proche du macroordinateur vers Azure

- [Micro Focus Visual COBOL](https://www.microfocus.com/products/visual-cobol/) outils de développement et d’intégration.
- [Micro Focus PL / je](https://www.microfocus.com/campaign/download/pli-modernization/) compilateur hérité pour la plate-forme .NET, en prenant en charge de macroordinateur PL / je syntaxe, les types de données et comportement.
- [Micro Focus Enterprise Server](https://www.microfocus.com/products/enterprise-suite/enterprise-server/) plate-forme d’intégration de macroordinateur.
- [Fujitsu NetCOBOL](https://www.fujitsu.com/global/products/software/developer-tool/netcobol/) outils de développement et d’intégration.
- [NTT données Enterprise COBOL](https://www.nttdataservices.com/en-ie/services/application-modernization-services) outils de développement et d’intégration.
- [NTT Open PL / je](https://www.nttdataservices.com/en-ie/services/application-modernization-services) compilateur hérité pour la plate-forme .NET, en prenant en charge de macroordinateur PL / je syntaxe, les types de données et comportement.
- [Compilateur Raincode COBOL](https://www.raincode.com/products/cobol/) outils de développement et d’intégration.
- [Raincode PL / je compilateur](https://www.raincode.com/products/pli/) pour .NET plateforme prend en charge mainframe PL / je syntaxe, les types de données et comportement.
- [Compilateur de Raincode ASM370](https://www.raincode.com/technical-landscape/asm370/) pour la syntaxe d’assembleur 370 et HLASM mainframe.
- [ASNA de RPG Visual pour .NET](https://asna.com/us/products/visual-rpg) compilateur pour le .NET Framework via le plug-in Visual Studio.
- [UCT systèmes modernes (COBOL à universel)](https://modernsystems.com/automatic-cobol-to-java-conversion/) outils de développement et d’intégration.

## <a name="deploy-an-emulation-environment-for-online-and-batch-processing"></a>Déployer un environnement d’émulation d’en ligne et de traitement par lots

- [Micro Focus Enterprise Server](https://www.microfocus.com/products/enterprise-suite/enterprise-server/) plate-forme d’intégration de macroordinateur.
- [Environnement de développement réhébergement NTT données Mainframe](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf) environnement de traitement de transaction native.
- [Environnement de traitement de lot de données NTT](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf) (BPE), y compris les fonctionnalités de langage JCL transaction.
- [Raincode CICS](https://www.raincode.com/technical-landscape/cics/) émulateur pour les plateformes .NET et Azure.
- [Raincode JCL](https://www.raincode.com/products/jcl/) interpréteur de langage JCL compatibles.

## <a name="code-conversion"></a>Conversion de code

- [Asysco](https://www.asysco.com/azure-cloud/) de source de conversion de code.
- [LzLabs logiciel défini Mainframe](https://www.lzlabs.com/) gérés conteneur de logiciel pour la migration d’applications mainframe vers des ordinateurs Linux ou private, public et les environnements de cloud hybride.
- [Âge de Blu](https://www.bluage.com/) outils de numérisation d’applications métier héritées et des bases de données.
- [Heirloom Computing](https://www.heirloomcomputing.com/tag/convert-cobol-to-java/) services pour convertir COBOL du macroordinateur CICS et VSAM de Java.
