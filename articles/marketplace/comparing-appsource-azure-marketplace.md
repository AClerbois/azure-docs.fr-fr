# <a name="comparing-appsource-and-the-azure-marketplace"></a>Comparaison entre AppSource et la Place de marché Azure
Les vitrines dans la [Place de marché Azure](https://azuremarketplace.microsoft.com) et [Microsoft AppSource](https://appsource.microsoft.com) répondent aux besoins spécifiques du client. Utilisez les vitrines pour cibler les clients par rôle. Vous pouvez offrir la solution ou le service approprié en fonction de votre client.

## <a name="understanding-the-differences-between-storefronts"></a>Différences entre les deux vitrines
Le choix d’une vitrine commence par l’identification du public cible pour votre offre. Si vous ciblez des publics dans plusieurs vitrines, vous pouvez effectuer une seule publication pour vendre dans plusieurs vitrines.
 
Les tableaux suivants décrivent les avantages de l’utilisation d’une vitrine :

### <a name="appsource"></a>AppSource

| Avantage | Détails | 
|:--- |:--- | 
| **Souplesse de facturation** | Provisionne une expérience d’évaluation. N’offre pas actuellement une option de publication activée pour le commerce. Vous pouvez utiliser votre infrastructure actuelle d’enregistrement des commandes et de facturation, sans autre modification ni investissement supplémentaire. | 
| **Associations avec les autres partenaires** | Associe les éditeurs de logiciels indépendants, les intégrateurs de systèmes et les fournisseurs de services managés à des scénarios d’implémentation spécifiques. Vous pouvez vendre collectivement à de nouveaux clients. | 
| **Automation** | Associe les éditeurs de logiciels indépendants, les intégrateurs de systèmes et les fournisseurs de services managés à des scénarios d’implémentation spécifiques. Vous pouvez vendre collectivement à de nouveaux clients. |
| **Plusieurs types de cloud** | Ne prend pas actuellement en charge Azure Stack, Azure Government et les clouds régionaux. | 
| **Présentation en contexte pour les clients** | Atteignez davantage de clients grâce à l’expérience dans l’application pour des produits Microsoft tels que Dynamics 365, Power BI et Office 365. |

### <a name="azure-marketplace"></a>Place de marché Azure

| Avantage | Détails | 
|:--- |:--- | 
| **Souplesse de facturation** | Pour les machines virtuelles, les options de facturation de type Paiement à l’utilisation sont basées sur des contrats Entreprise Microsoft ou des modèles de vente directe sur le web. Les options de tarification incluent un abonnement de niveau Gratuit où une offre est gratuite en permanence. Les options de tarification incluent également un abonnement Essayer maintenant. Un abonnement Try It Now (Essayer maintenant) est disponible gratuitement pour une période limitée. À la fin de la période, la machine virtuelle est convertie en un abonnement payant. L’activation BYOL (Apportez votre propre licence) est également une option que vous pouvez utiliser pour les clients.<br /><br/>Pour les deux options de facturation et les offres de machine virtuelle suivantes, toutes les ressources Azure provisionnées sont facturées directement au client :<ul> <li>Applications Azure : application gérée</li> <li>Applications Azure : modèle de solution</li> </ul> | 
|**Associations avec les autres partenaires** | Actuellement, vous ne pouvez pas lier un fournisseur de services ou des partenaires de livraison à votre offre. | 
| **Automation** | Actuellement, vous ne pouvez pas lier un fournisseur de services ou des partenaires de livraison à votre offre. Tirez parti d’applications SaaS automatisées avec le provisionnement de compléments. Utilisez des applications Azure : option de publication d’un modèle de solution pour automatiser la collecte de données SaaS et les scénarios de déploiement. |
| **Plusieurs types de cloud** | Publiez des solutions pour le cloud public et au niveau local à l’aide des types suivants :<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>Clouds régionaux, notamment en Chine et en Allemagne</li></ul> | 
| **Présentation en contexte pour les clients** | Mettez votre solution à disposition dans le portail Azure pour que les clients puissent effectuer une recherche contextuelle. Utilisez les applications de machine virtuelle et Azure : options de publication de modèle de solution. |
