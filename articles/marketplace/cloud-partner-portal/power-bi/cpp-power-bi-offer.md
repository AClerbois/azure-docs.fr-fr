# <a name="power-bi-application-offer"></a>Offre d’application Power BI

|                  |                                |
|------------------|--------------------------------|
| Cette section explique comment publier une application Power BI sur la [Place de marché Microsoft AppSource](https://appsource.microsoft.com/). Package d’application Power BI et contenu Power BI utilisant des flux de données pour relier les rapports et tableaux de bord aux données dans l’espace de stockage des données communes. L’application peut ensuite être déployée vers d’autres locataires par le biais d’AppSource. | ![Icône Power BI](./media/powerbi-icon.png) |


Cette section comprend les parties principales suivantes :

-   [Conditions préalables](./cpp-prerequisites.md) : exigences techniques et commerciales pour la création ou la publication d’une offre d’application Power BI
-   [Créer l’offre d’application Power BI](./cpp-create-offer.md) : étapes requises pour créer une offre d’application Power BI à l’aide du [portail Microsoft Cloud Partner](https://cloudpartner.azure.com)
-   [Publier une offre d’application Power BI](./cpp-publish-offer.md) : procédure de soumission d’une nouvelle offre en vue de sa publication sur AppSource, et procédure de mise à jour d’une offre existante.


## <a name="publishing-steps"></a>Étapes de publication

Le schéma suivant illustre les étapes principales de la publication d’une offre d’application Power BI.

![Étapes de publication d’application Power BI](media/publishing-steps.png)

Processus de publication d’application Power BI

1. Créez une application de service dans Power BI (ce qui se traduit par la création d’une URL d’installation de package).  Cette URL représente les ressources techniques de l’offre.  Pour plus d’informations, consultez [Qu’est-ce que Power BI](https://go.microsoft.com/fwlink/?linkid=2028636).
2. Collecter ou créer les ressources d’entreprise, notamment :
    - Supports marketing
    - Documents juridiques et de support
3.  Créer l’offre : utilisez le Portail Cloud Partner pour configurer les détails et les informations sur l’offre, y compris la description de l’offre, les documents marketing, les références juridiques, les informations de support et les spécifications des ressources.
4.  Certifier l’offre et la publier : cette soumission commence le processus de publication.
    - L’équipe d’intégration AppSource teste, valide et certifie votre application. 
    - Une fois certifiée, vous devez la passer en revue dans son environnement de test et la libérer.

Une fois toutes ces étapes effectuées, votre application démarre sur AppSource.

## <a name="next-steps"></a>Étapes suivantes

Avant de commencer ces étapes, vous devez respecter les [exigences](./cpp-prerequisites.md) de la publication d’une application Power BI sur AppSource.
