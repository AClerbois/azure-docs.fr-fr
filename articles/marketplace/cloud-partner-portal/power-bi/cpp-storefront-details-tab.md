# <a name="power-bi-app-storefront-details-tab"></a>Onglet Détails de la vitrine des applications Power BI

Sur la page **Nouvelle offre**, utilisez l’onglet **Détails de la vitrine** pour fournir des informations légales, commerciales et marketing à vos clients potentiels. Dans cet onglet, vous pouvez également configurer la gestion des prospects générés par la Place de marché Azure. Ce formulaire long est divisé en six sections : **Détails de l’offre**, **Détails de l’annonce**, **Artefacts marketing**, **Informations légales**, **Service clientèle** et **Gestion des prospects**.  Un astérisque (*) à la fin d’une étiquette de champ signifie que le champ est obligatoire.


## <a name="offer-details-section"></a>Section Détails de l’offre

Dans la section **Détails de l’offre** section, entrez des informations générales sur votre offre AppSource.

![Section Détails de l’offre de l’onglet Détails de la vitrine](./media/offer-details-section.png)

Le tableau suivant fournit des informations supplémentaires sur ces champs. Les champs obligatoires sont indiqués par un astérisque (*).  

|   Champ               |   Description                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **Résumé de l’offre\***     | Une brève description de l’objet de l’application comptant 100 caractères au maximum.                             |
| **Description de l’offre\*** | Une description de l’application comptant 3 000 caractères au maximum. Ce champ prend en charge la mise en forme HTML simple. |
|   |    |


## <a name="listing-details-section"></a>Section Détails de l’annonce

La section **Listing Details** (Détails de l’annonce) rassemble des informations sur le contexte de votre application : secteurs où elle est généralement utilisée, catégorie d’application, produits compatibles et termes de recherche associés.

![Section Détails de l’annonce de l’onglet Détails de la vitrine](./media/listing-details-section.png)

Pour renseigner les champs figurant dans cette section, utilisez le tableau suivant.  Les champs obligatoires sont indiqués par un astérisque (*).
 
|   Champ                                  |   Description                                                        |
| --------------                           | ---------------------                                                |
| **Secteurs d’activité**                           | Sélectionnez le secteur d’activité correspondant le mieux à votre application. Si votre application est liée à plusieurs secteurs d’activité, laissez ce champ vide.      |
| **Catégories\***                           | Sélectionnez jusqu’à trois catégories qui se rapportent à votre application.     |
| **Lien d’aide pour votre application\***               | Indiquez une URL vers une page offrant une aide en ligne pour votre application.           |
| **Produits avec lesquels votre application fonctionne (3 max.)** | Sélectionnez l’option **Nouveau** précédée du signe plus pour créer un champ de texte. Dans le champ, entrez le nom d’un produit avec lequel votre application fonctionne. Indiquez jusqu’à trois produits.       |
| **Mots clés de recherche (3 max.)**              | Entrez jusqu’à trois mots clés que les utilisateurs sont susceptibles d’utiliser pour rechercher votre application dans AppSource. Par exemple, si votre application est une application de messagerie, vous pouvez entrer les mots clés **e-mails**, **messages électroniques** et **application de courrier**. |
|  |  |


## <a name="marketing-artifacts-section"></a>Section Artefacts marketing

Dans la section **Marketing Artifacts** (Artefacts marketing) chargez les documents de personnalisation et marketing à afficher sur AppSource.  Cette section comprend quatre sous-sections : **Logos**, **Vidéos**, **Documents** et **Captures d’écran**. Les logos et les captures d’écran sont les seuls artefacts marketing requis . Cependant, pour attirer plus efficacement les clients, nous vous recommandons d’ajouter des vidéos et des documents.

![Section Artefacts marketing de l’onglet Détails de la vitrine](./media/marketing-artifacts-section.png)

Pour renseigner les champs figurant dans cette section, utilisez le tableau suivant. Les champs obligatoires sont indiqués par un astérisque (*).
 
|    Champ                             |    Description                                                    |
|   -----------                        |    -------------                                                  |
| *Logos*                              |                                                                   |
| **Logo de l’offre (format png, 48 x 48)\***   | Chargez un logo à afficher dans la vue d’ensemble de l’application ou dans les résultats de recherche d’application. AppSource prend en charge uniquement le format PNG avec une résolution de 48 x 48 px.  |
| **Logo de l’offre (format png, 216 x 216)\*** | Chargez un logo à afficher sur la page des informations de votre application.  AppSource prend en charge uniquement le format PNG avec une résolution de 216 x 216 px.  |
| *Vidéos*                             |                                                                   |
| **Nom**                             | Entrez le nom ou le titre de votre application.                                          |
| **URL**                              | Entrez l’URL d’une vidéo hébergée sur YouTube ou Vimeo.                              |
| **Vidéo miniature**                        | Ajoutez une image miniature de l’application.  AppSource prend en charge uniquement le format PNG avec une résolution de 1 280 x 720 px.   |
| *Documents*                          | Ajoutez jusqu’à trois documents à afficher sur AppSource, sous le titre **En savoir plus**.  |
| **Nom**                             | Entrez le nom ou le titre d’un document de prise en charge.                              |
| **File**                             | Chargez un fichier PDF.                             |
| *Captures d’écran\**                      | Ajoutez jusqu’à cinq captures d’écran.                        |
| **Nom**                             | Entrez le nom ou le titre de d’une capture d’écran.                                       |
| **Image**                            | Chargez une capture d’écran PNG. Sa résolution doit être 1 280 x 720 px.  | 
|   |   |

Les logos que vous chargez sur le [Portail Cloud Partner](https://cloudpartner.azure.com) ne doivent *pas* :

- Utiliser un dégradé. L’apparence de votre logo doit être « plate ».
- Inclure le nom de votre société, un nom de marque ou un autre texte. 
- Sembler étiré.

## <a name="legal-section"></a>Section Informations légales

Dans la section **Informations légales**, fournissez les deux documents juridiques nécessaires pour chaque offre : la politique de confidentialité et les conditions d’utilisation.

![Section Informations légales de l’onglet Détails de la vitrine](./media/legal-section.png)

Pour renseigner les champs figurant dans cette section, utilisez le tableau suivant :

|   Champ                |   Description                           |
|------------------------|--------------------------------------   |
| **URL de la politique de confidentialité\*** | URL de votre politique de confidentialité publiée       |
| **Conditions d’utilisation\***       | Votre politique d’utilisation, sous forme de texte brut ou HTML simple     |
|  |  |


## <a name="customer-support-section"></a>Section Service clientèle

Dans la section **Service clientèle**, indiquez **l’URL du support** permettant d’accéder à votre page de support en ligne.  Cette page doit fournir plusieurs options de contact, par exemple un numéro de téléphone, une adresse e-mail et un lien de conversation en direct. 


## <a name="lead-management-section"></a>Section Gestion des prospects

Dans la section **Gestion des prospects**, configurez votre système pour collecter les prospects générés par vos offres AppSource. Pour choisir vos options de stockage de prospects, utilisez le tableau suivant :

|    Champ               |   Destination du prospect                               |
|------------------------|--------------------------------------            |
|  **Aucun**              | Aucun prospect n’est collecté. Il s'agit de l'option par défaut.  |
| **Objet blob Azure (déprécié)** | Spécifiez le [Stockage Blob Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview) en utilisant un nom de conteneur et une chaîne de connexion.  Ce choix est déconseillé. Utilisez plutôt un stockage **Table Azure**.  |
| **Table Azure**        | Spécifiez un [stockage Table Azure](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview) à l’aide d’une chaîne de connexion.  |
| **Dynamics CRM Online** | Spécifiez [Dynamics 365](https://dynamics.microsoft.com/) en utilisant une URL et des informations d’identification pour l’authentification. |
| **Point de terminaison HTTPS**     | Spécifiez le point de terminaison HTTPS en utilisant une charge utile JSON.   |
| **Marketo**            | Spécifiez une instance [Marketo](https://www.marketo.com/) en utilisant un ID de serveur, un ID Munchkin et un ID de formulaire.   |
| **Salesforce**         | Spécifiez [Salesforce](https://www.salesforce.com/) en utilisant un identificateur d’objet. |
|  |  |

Une fois votre offre publiée, la connexion aux prospects est vérifiée et un prospect test est automatiquement envoyé à la destination spécifiée. Générez les informations relatives aux prospects en continu, et mettez vos paramètres à jour rapidement pour refléter votre architecture actuelle de gestion des clients.
