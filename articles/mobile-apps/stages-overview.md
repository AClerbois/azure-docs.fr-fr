# <a name="different-stages-in-mobile-application-development"></a>Différentes phases du développement d’applications mobiles
La création d’une application mobile implique plusieurs phases, et les développeurs peuvent facilement tirer parti des services, outils et technologies Microsoft pour créer des applications mobiles de grande qualité.

## <a name="app-development"></a>Développement d’applications
Les développeurs peuvent créer des applications clientes natives à plateforme unique avec des langages comme Objective-C et Java, des applications multiplateformes à l’aide de Xamarin, React Native et Unity, et des applications hybrides à l’aide de Cordova. Ils peuvent utiliser des IDE et des éditeurs de code, tels que Visual Studio, Visual Studio pour Mac ou Visual Studio Code, sur des PC et des Mac, pour construire leurs applications clientes.

## <a name="plan-and-design-with-back-end-services"></a>Planifier et concevoir avec des services back-end
 Microsoft fournit divers services pour la configuration d’un back-end sécurisé, scalable et fiable pour votre application mobile. Les développeurs peuvent créer des API et des déclencheurs serverless sans avoir à tenir à jour leurs propres serveurs. Ils peuvent tirer parti de services tels que les notifications push, la synchronisation des données hors connexion, l’authentification et les données, qui ensemble forment un back-end mobile complet et étendu en tant que service qui prend en charge un large éventail de scénarios de construction d’applications. La création d’un service back-end présente de nombreuses exigences :
   - Limite de stockage des données sur l’appareil
   - Synchronisation des données sur plusieurs appareils
   - Garantie que l’application peut s’exécuter sans connectivité réseau
   - Envoi de notifications aux utilisateurs
   - Réduction de la consommation de la batterie
   - Création d’un back-end scalable, fiable et sécurisé
   - Maintenance de serveur automatique

## <a name="devops-and-continuous-monitoring"></a>DevOps et supervision continue
Les développeurs qui créent des applications mobiles et basées sur le cloud peuvent tirer parti des services d’intégration continue (CI), de livraison continue (CD) et de supervision continue afin de produire des builds natives pour toutes les plateformes, d’exécuter des tests d’automatisation de l’interface utilisateur sur des milliers d’appareils physiques, d’automatiser leurs mises en production par le biais de canaux de test bêta, ou de déployer directement sur des magasins d’applications. De plus, ils peuvent recueillir des informations sur les plantages et les exceptions, ainsi que des données d’analyse de l’utilisation pour leurs applications.

## <a name="additional-services"></a>Services supplémentaires
De nombreux services Azure peuvent être utilisés, en fonction du type et de la nature de l’application en cours de génération.
  - Recherche Azure pour la recherche de tous les types de données internes, avec interrogation rapide
  - Azure Cognitive Services pour utiliser l’intelligence artificielle dans votre application
  - Azure SignalR pour activer la communication en temps réel dans votre application
