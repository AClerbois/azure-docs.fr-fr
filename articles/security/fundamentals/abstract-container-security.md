# <a name="container-security-in-microsoft-azure"></a>Sécurité des conteneurs dans Microsoft Azure
## <a name="abstract"></a>Résumé

La technologie de conteneur a engendré des changements structurels dans le monde du cloud computing. Les conteneurs permettent d’exécuter plusieurs instances d’une application sur une instance unique d’un système d’exploitation, améliorant ainsi l’utilisation des ressources. Les conteneurs offrent de la flexibilité et de la cohérence aux organisations. Ils permettent un déploiement continu, car l’application peut être développée sur un ordinateur de bureau, testée sur une machine virtuelle, puis déployée pour la production dans le cloud. Les conteneurs offrent une souplesse, une rationalisation des opérations, une scalabilité et une réduction des coûts grâce à l’optimisation des ressources.

La technologie de conteneur étant relativement nouvelle, de nombreux professionnels de l’informatique ont quelques inquiétudes relatives à la sécurité, sur le manque de visibilité et l’utilisation dans un environnement de production. Les équipes de développement n’ont souvent pas connaissance des bonnes pratiques de sécurité. Ce livre blanc peut aider les équipes chargées des opérations de sécurité et les développeurs à sélectionner des approches permettant de sécuriser le développement et le déploiement de conteneurs sur la plateforme Microsoft Azure.

Il décrit les conteneurs, le déploiement et la gestion des conteneurs, ainsi que les services de plateforme natifs. Il décrit également les problèmes de sécurité d’exécution qui se posent lors de l’utilisation de conteneurs sur la plateforme Azure. Dans les figures et les exemples, ce document utilise Docker comme modèle de conteneur et Kubernetes comme orchestrateur de conteneurs. La plupart des recommandations de sécurité s’appliquent également aux autres modèles de conteneurs fournis par les partenaires de Microsoft sur la plateforme Azure.
