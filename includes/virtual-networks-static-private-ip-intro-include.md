# Adresse IP interne statique

Vos machines virtuelles IaaS et instances de rôle PaaS dans un réseau virtuel reçoivent automatiquement une adresse IP interne à partir d'une plage que vous spécifiez. Cette adresse est conservée par les machines virtuelles et les instances de rôle, jusqu'à ce qu'elles soient mises hors service. Vous mettez une machine virtuelle ou une instance de rôle hors service en l'arrêtant à partir de PowerShell, l'interface de ligne de commande Azure ou le portail Azure. Dans ce cas, lorsque la machine virtuelle ou l'instance de rôle redémarre, elle reçoit une adresse IP disponible à partir de l'infrastructure Azure, qui peut ne pas être identique à celle qu'elle avait précédemment. Dans certains cas, vous ne souhaitez pas qu'une machine virtuelle ou une instance de rôle ait une adresse IP dynamique, par exemple, si votre machine virtuelle doit exécuter DNS ou fera office de contrôleur de domaine.

>[AZURE.NOTE]Une adresse IP interne dynamique reste associée à la machine virtuelle si vous arrêtez la machine virtuelle à partir du système d'exploitation invité. Les machines virtuelles sont uniquement mises hors service lorsqu'elles sont arrêtées à partir du portail Azure, de PowerShell ou de l'interface de ligne de commande Azure.
