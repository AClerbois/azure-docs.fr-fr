# Gestion des données : compréhension des technologies NoSQL sur Azure

**Résumé :** Azure propose un vaste choix d'options pour vos charges de travail NoSQL, mais il peut être difficile de déterminer celle qui vous convient. Ce document présente les technologies NoSQL sur Azure et explique comment choisir le service approprié pour gérer vos données non relationnelles, qui incluent des ensembles de données non structurés ou particulièrement volumineux, souvent désignés par le terme « Big Data ».

La technologie relationnelle a été pendant longtemps l'approche prédominante pour l'exploitation des données. Toutefois, lorsque vous utilisez de très grosses quantités de données de types différents, la mise à l'échelle sur de nombreux serveurs est une opération difficile. De plus, la technologie relationnelle n'est pas la meilleure solution pour gérer certains types de données tels que les documents JSON ou les graphiques. Ce guide passe en revue les différentes options, en décrivant le rôle de chacune et en expliquant dans quelles circonstances vous pouvez les utiliser.
