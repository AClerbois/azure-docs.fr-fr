# <a name="comparing-appsource-and-azure-marketplace"></a>Comparaison entre AppSource et la Place de marché Azure  
Les vitrines servent les besoins spécifiques du client. Les vitrines permettent le ciblage par rôle. Vous pouvez offrir la solution ou le service approprié en fonction de votre client.  

## <a name="understanding-the-differences-between-the-storefronts"></a>Comprendre les différences entre les vitrines  
Le choix d’une vitrine commence par l’identification du public cible pour votre offre. Si vous ciblez les deux publics, vous pouvez alors publier une seule fois pour vendre dans les deux vitrines.  
 
Prenez en compte les autres avantages des vitrines.   

### <a name="appsource"></a>AppSource  

| Avantages | Détails | 
|:--- |:--- | 
| ***Souplesse de facturation*** | Provisionne une expérience d’évaluation. N’offre pas actuellement une option de publication activée pour le commerce. Vous pouvez utiliser votre infrastructure actuelle d’enregistrement des commandes et de facturation, sans autre modification ni investissement supplémentaire. | 
| ***Simplification des mises en relation avec d’autres partenaires*** | Associe les éditeurs de logiciels indépendants, les intégrateurs de systèmes et les fournisseurs de services managés à des scénarios d’implémentation spécifiques. Vous pouvez vendre collectivement à de nouveaux clients. | 
| ***Automation*** | Associe les éditeurs de logiciels indépendants, les intégrateurs de systèmes et les fournisseurs de services managés à des scénarios d’implémentation spécifiques. Vous pouvez vendre collectivement à de nouveaux clients. |  
| ***Plusieurs types de cloud*** | Ne prend pas actuellement en charge Azure Stack, Azure Government et les clouds régionaux. | 
| ***Présentation en contexte pour les clients*** | Atteignez davantage de clients grâce à l’expérience dans l’application pour des produits Microsoft tels que Dynamics 365, Power BI et Office 365. |  

### <a name="azure-marketplace"></a>Place de marché Azure  

| Avantages | Détails | 
|:--- |:--- | 
| ***Souplesse de facturation*** | Pour les machines virtuelles, les options de facturation de type Paiement à l’utilisation sont basées sur des contrats Entreprise Microsoft ou des modèles de vente directe sur le web. Les options de tarification incluent un abonnement de niveau Gratuit où une offre est gratuite en permanence. Les options de tarification incluent également un abonnement Essayer maintenant. Un abonnement de type Essayer maintenant reste gratuit pendant une période limitée, et à l’issue de celle-ci, la machine virtuelle devient un abonnement payant. L’activation BYOL est également une option permettant de vous soutenir.<br />Pour les deux options de facturation et les offres de machine virtuelle suivantes, toutes les ressources Azure provisionnées sont facturées directement au client.<ul> <li>Applications Azure : application gérée</li> <li>Applications Azure : modèle de solution</li> </ul> | 
| ***Simplification des mises en relation avec d’autres partenaires*** | Actuellement, vous ne pouvez pas lier un fournisseur de services ou des partenaires de livraison à votre offre. Cette fonctionnalité sera lancée en 2018. | 
| ***Automation*** | Actuellement, vous ne pouvez pas lier un fournisseur de services ou des partenaires de livraison à votre offre. Tirez parti d’applications SaaS automatisées avec le provisionnement de compléments. Utilisez des applications Azure : modèle de solution pour automatiser la collecte de données SaaS et les scénarios de déploiement. |  
| ***Plusieurs types de cloud*** | Publiez sur le cloud public et les solutions locales à l’aide des types suivants.<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>clouds régionaux, notamment en Chine et en Allemagne</li> </ul> | 
| ***Présentation en contexte pour les clients*** | Présentez votre solution dans le portail Azure pour que les clients puissent effectuer une recherche contextuelle à l’aide des machines virtuelles et des applications Azure : modèle de solution. |
